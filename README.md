## 01. 💻 컴퓨터 구조 (Computer Architecture)

### 1) 컴퓨터가 이해하는 두 가지 핵심 정보

> 컴퓨터는 오직 명령어를 실행하고, 그에 필요한 데이터를 처리할 뿐입니다.
> 
- **⚙️ 명령어**: 컴퓨터를 실질적으로 움직이는 실행 정보.
- **📦 데이터**: 명령어가 수행되기 위한 재료가 되는 정보.

---

### 2) 컴퓨터의 4가지 핵심 부품

컴퓨터 내부의 핵심적인 역할을 수행하는 부품들에 대해 알아봅니다.

| 부품 | 아이콘 | 설명 및 역할 |
| --- | --- | --- |
| **CPU** (중앙처리장치) | 🧠 | **컴퓨터의 두뇌**. 메모리의 명령어를 읽고, 해석하고, 실행합니다. |
| **메모리** (주기억장치) | 💾 | 현재 실행되는 프로그램의 명령어와 데이터를 저장하는 공간입니다. |
| **보조기억장치** | 💽 | 전원이 꺼져도 내용을 보관할 프로그램을 반영구적으로 저장합니다. (예: HDD, SSD) |
| **입출력장치** | ⌨️📺 | 컴퓨터 외부에서 정보를 교환하는 부품들입니다. (예: 키보드, 모니터) |

### 💡 CPU 내부의 핵심 구성 요소

- **제어장치 (Control Unit)**: 제어 신호를 생성하고 명령어를 해석합니다.
- **ALU (Arithmetic Logic Unit)**: 사칙연산 및 논리 연산을 수행하는 계산기입니다.
- **레지스터 (Register)**: CPU 내부의 아주 작고 빠른 임시 저장 공간입니다.

> **잠깐! 메모리 핵심 포인트**
> 
> - 프로그램이 실행되려면 **반드시 메모리에 저장**되어 있어야 합니다.
> - 메모리에 저장된 값의 위치는 **주소(Address)**로 알 수 있습니다.

---

### 3) 메인보드와 시스템 버스

모든 핵심 부품은 **메인보드**에 연결되어 있으며, 데이터를 주고받기 위한 통로인 **시스템 버스**를 사용합니다.

### 🚌 시스템 버스의 세 가지 통로

- **주소 버스**: 주소를 주고받는 통로.
- **데이터 버스**: 명령어와 데이터를 주고받는 통로.
- **제어 버스**: 제어 신호를 주고받는 통로.

## 02. 🔢 데이터 (Data Representation)

### 1) 숫자의 표현 방법: 0과 1

컴퓨터는 모든 숫자를 **이진법**과 **십육진법**으로 표현합니다.

- **워드 (Word)**: CPU가 한 번에 처리할 수 있는 데이터의 기본 단위.
- **이진법**: 0과 1로만 수를 표현하는 방법.
- **십육진법**: 0~9, A~F까지 사용하여 수를 표현하는 방법. 이진수와의 상호 변환이 용이합니다.

### 2) 문자의 표현 방법: 0과 1

컴퓨터가 문자를 이해하는 과정을 살펴봅니다.

- **인코딩 (Encoding)**: 문자를 0과 1로 이루어진 문자 코드로 변환하는 과정.
- **디코딩 (Decoding)**: 0과 1로 표현된 문자 코드를 다시 문자로 변환하는 과정.

### 🔤 주요 문자 인코딩 방식

- **ASCII**: 초창기 7비트 문자 집합 (알파벳, 숫자, 일부 특수문자). 오류 검출용 패리티 비트를 포함하여 8비트로 사용하기도 함.
- **EUC-KR**: 한글을 위한 인코딩 방식. 한 글자당 2바이트를 부여하는 **완성형** 인코딩.
- **유니코드 (Unicode)**: 통일된 문자 집합. 한글, 영어, 특수 문자, 이모티콘까지 포함하는 전 세계 공통 표준. 현대 문자 표현에 핵심적.
- **UTF-8, UTF-16**: 유니코드 데이터를 인코딩하는 가변 길이 방식들.

## 03. 🛠️ 명령어 (Instruction)

### 1) 언어의 분류

컴퓨터가 이해하는 언어와 인간이 이해하는 언어의 차이.

- **고급 언어**: 사람이 이해하기 쉬운 언어. (예: Python, Java, C++)
- **저급 언어**: 컴퓨터가 직접 이해하는 언어.
    - **기계어**: 0과 1로만 이루어진 언어.
    - **어셈블리어**: 기계어를 사람이 이해할 수 있는 문자 기호로 바꾼 언어.

### 🔄 변환 방식에 따른 분류

- **컴파일 (Compile)**: 소스 코드 전체를 기계어로 변환한 뒤 실행. (속도가 빠름)
- **인터프리터 (Interpret)**: 소스 코드를 한 줄씩 기계어로 변환하며 실행. (수정이 용이)

---

### 2) 명령어의 구조

> 명령어는 무엇을 할 것인지(**연산 코드**)와 무엇을 대상으로 할 것인지(**오퍼랜드**)로 구성됩니다.
> 

$$\text{명령어} = \text{연산 코드} + \text{오퍼랜드}(0 \sim \text{여러 개})$$

### 📋 주요 연산 코드 (Opcode)

| 유형 | 대표적인 연산 | 설명 |
| --- | --- | --- |
| **데이터 전송** | `MOVE`, `STORE`, `LOAD`, `PUSH`, `POP` | 데이터를 이동, 저장, 가져오거나 스택에 넣고 뺍니다. |
| **산술/논리** | `ADD`, `SUB`, `AND`, `OR` | 사칙연산 및 논리 연산을 수행합니다. |
| **제어 흐름** | `JUMP`, `CALL`, `RETURN` | 프로그램 실행 순서를 변경합니다. |
| **입출력 제어** | `INPUT`, `OUTPUT` | 외부 장치와 정보를 주고받습니다. |

### 🎯 오퍼랜드와 주소 지정 방식 (Addressing Modes)

명령어는 '무엇을 대상으로' 연산을 수행할지 명확히 해야 합니다. 이때 사용되는 데이터나 그 위치를 다루는 방식들입니다.

### 1) 핵심 용어 정의

- **오퍼랜드 (Operand)**: 연산에 사용할 **데이터** 그 자체 또는 데이터가 저장된 **위치(메모리 주소, 레지스터 이름)**를 의미합니다.
- **유효 주소 (Effective Address)**: 연산에 사용할 데이터가 실제로 저장된 최종적인 메모리 주소를 말합니다.

---

### 2) 주소 지정 방식의 종류

데이터를 찾는 방식에 따라 성능과 주소 표현 범위가 달라집니다.

| 방식 명칭 | 작동 원리 | 특징 |
| --- | --- | --- |
| **즉시 주소 지정**<br>(Immediate) | 오퍼랜드 필드에 **실제 데이터**를 직접 명시합니다. | 별도의 메모리 접근이 없어 가장 빠르지만, 표현할 수 있는 데이터 크기에 제한이 있습니다. |
| **직접 주소 지정**<br>(Direct) | 오퍼랜드 필드에 **유효 주소(메모리 주소)**를 직접 명시합니다. | 유효 주소를 찾는 과정이 간단하지만, 오퍼랜드 필드의 비트 수만큼만 주소를 표현할 수 있습니다. |
| **간접 주소 지정**<br>(Indirect) | 오퍼랜드 필드에 **유효 주소의 주소**를 명시합니다. | 직접 주소 지정보다 훨씬 넓은 주소 범위를 표현할 수 있지만, 메모리 접근이 최소 2번 발생하여 속도가 느립니다. |
| **레지스터 주소 지정**<br>(Register) | 연산에 사용할 데이터를 저장한 **레지스터**를 오퍼랜드 필드에 직접 명시합니다. | 메모리가 아닌 CPU 내부의 레지스터를 참조하므로 직접 주소 지정 방식보다 속도가 매우 빠릅니다. |
| **레지스터 간접 주소 지정**<br>(Register Indirect) | **유효 주소를 저장하고 있는 레지스터**를 오퍼랜드 필드에 명시합니다. | 주소는 메모리에 있지만 그 주소를 레지스터가 들고 있는 형태입니다. 메모리 접근 횟수를 줄이면서도 간접 주소 지정의 장점을 챙길 수 있습니다. |
